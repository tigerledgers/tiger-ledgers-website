Software migrations fail for predictable reasons: unclear scope, missing data lineage, and no rollback plan. Here's how we run them.

## Phase 1: Map the current state

Document every report, every recurring journal entry, and every integration. This is unglamorous and absolutely required.

## Phase 2: Cut over on a clean fiscal boundary

Always migrate at month-end or year-end. Mid-period migrations create reconciliation nightmares.

## Phase 3: Run parallel for one cycle

Both systems run for one full close cycle. Differences are reconciled before the legacy system is shut down.

A clean migration is invisible to the business. That's the entire goal.
